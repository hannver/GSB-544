def create_pipeline(degree):
    return Pipeline([
        ('poly', PolynomialFeatures(degree=degree)),
        ('ols', LinearRegression())
    ])

linear_model = create_pipeline(1)
linear_model.fit(X, y)
y_linear = linear_model.predict(X)

quadratic_model = create_pipeline(2)
quadratic_model.fit(X, y)
y_quadratic = quadratic_model.predict(X)

cubic_model = create_pipeline(3)
cubic_model.fit(X, y)
y_cubic = cubic_model.predict(X)

degree10_model = create_pipeline(10)
degree10_model.fit(X, y)
y_degree10 = degree10_model.predict(X)
sorted_indices = X.squeeze().argsort()
X_sorted = X.iloc[sorted_indices]
y_sorted = y.iloc[sorted_indices]

y_linear_sorted = y_linear[sorted_indices]
y_quadratic_sorted = y_quadratic[sorted_indices]
y_cubic_sorted = y_cubic[sorted_indices]
y_degree10_sorted = y_degree10[sorted_indices]

plt.scatter(X_sorted, y_sorted, color='blue', label='Data')

  • Are any of the models above underfitting the data? If so, which ones and how can you tell?
  • Are any of thhe models above overfitting the data? If so, which ones and how can you tell?
  • Which of the above models do you think fits the data best and why?

The data appear to be underfitted by the linear model. When a model is unable to precisely represent the relationship between the input(s) and the output variable, it is said to be underfitting. Because the linear model just uses one straight line to describe the data rather than capturing the curves contained in the data, it is underfitting. The data is being overfitted by the Degree 10 model. When a model remembers too much about the training set of data, it is considered to be overfitting. In other words, the model is accounting for both the relationship between the input or inputs and the output as well as some of the subtleties or noise in the data. The Cubic model is the best fit for the datasetaccording to the figure. Without being overly basic, it plots the data's overall trend and curves.Although the Cubic model plots to be a marginally better match, the Quadratic model also matches the data quite well.
